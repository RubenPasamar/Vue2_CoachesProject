# Vue2-testing

Utilizaremos este repositorio para hacer test sobre componentes y store de VUE2.

# Utilización del repo

- La rama `main` es la rama principal del proyecto. Solo tenéis que utilizarla para iniciar vuestra rama desde ese punto y actualizar cuando consideréis.
- El proyecto está sin terminar, se irá actualizando, por lo que nos puede servir de ejercicio para una vez que hayamos creado test que testeen el 100% del componente tengamos que modificarlos debido a cambios en la estructura del store, en el propio componente, etc. Este ejercicio es interesante porque en el dia a día de desarrollo web puede suceder.
- Si queréis podéis utilizar el propio repo para subir vuestros cambios pero para eso es **imprescindible** que cada uno cree una rama con su nombre para evitar interferir en el código de los demás. Además, cada cierto tiempo intentaría mergear `main` en mi rama y lanzar de nuevo los tests para ver que todo sigue funcionando y solucionarlo en caso de que no sea así.
